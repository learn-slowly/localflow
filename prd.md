# LocalFlow — 지역 선거 유동인구 분석 플랫폼

> **프로젝트명**: LocalFlow **목적**: 기초의원 선거 유세 전략 수립을 위한 유동인구·교통·상권·선거 데이터 종합 분석 플랫폼 **핵심 설계 원칙**: 모든 데이터를 API로 수집하여, 도시코드만 교체하면 전국 어떤 시·군에도 적용 가능한 구조 **작성일**: 2026-03-25 **작성자**: 레고

---

## 1. 프로젝트 배경

- 2026년 6월 3일 제9회 전국동시지방선거 예정
- 진주시 기초의원(시의원) 선거에서 "어디에, 언제 사람이 모이는가"를 데이터로 파악
- 정의당 경남도당 차원의 유세 전략 수립 근거 자료
- **확장 목표**: 성공하면 경남 내 다른 시·군, 나아가 전국 적용

## 2. API 기반 데이터 아키텍처

### 2.1 설계 원칙

```
[설정 파일]          [API 레이어]           [시각화]
cityCode: 38030  →  각 API에 도시코드 전달  →  지도 렌더링
cityName: 진주시     데이터 정규화           레이어 토글
```

- 도시코드(행정표준코드) 하나로 모든 API 호출을 통일
- 진주시 = 38030 (시군구 코드 5자리)
- 다른 도시 적용 시 설정 파일의 코드만 변경

## 3. 확장 가능한 시스템 구조

### 3.1 설정 파일 (config)

```typescript
// config/city.ts
export interface CityConfig {
  code: string;        // 시군구 코드 5자리 (예: "38030")
  name: string;        // 도시명 (예: "진주시")
  province: string;    // 시도명 (예: "경상남도")
  center: [number, number]; // 지도 중심 좌표 [위도, 경도]
  zoom: number;        // 기본 줌 레벨
  tagoCode: number;    // TAGO 도시코드 (별도 매핑 필요)
  electionId: string;  // 선관위 선거ID
}

export const JINJU: CityConfig = {
  code: "38030",
  name: "진주시",
  province: "경상남도",
  center: [35.1798, 128.1076],
  zoom: 12,
  tagoCode: 38030,  // TAGO 도시코드 목록 API로 확인 필요
  electionId: "20220601", // 제8회 지방선거
};
```

### 3.2 API 래퍼 구조

```
src/lib/api/
├── config.ts          # API 키, 베이스 URL 관리
├── population.ts      # ① 인구 데이터
├── commerce.ts        # ② 상권 데이터
├── election.ts        # ③④⑤⑥ 선거 데이터 (투개표, 선거인수, 투표소, 코드/후보자)
├── boundary.ts        # ⑦ 행정경계 GeoJSON (safetydata.go.kr)
├── transport.ts       # ⑧ 교통카드 승하차·버스 데이터 (stcis.go.kr)
└── types.ts           # 공통 타입 정의
```

각 모듈은 도시코드(CityConfig)를 받아서 해당 도시의 데이터를 반환. → 새 도시 추가 = 새 CityConfig 객체 하나만 만들면 끝.

### 3.4 기술 스택

| 구분 | 선택 | 이유 |
| --- | --- | --- |
| 프레임워크 | Next.js 15 + TypeScript | gasout 동일 스택 |
| 스타일링 | Tailwind CSS | 빠른 UI |
| 지도 | Leaflet + react-leaflet | gasout 검증 완료 |
| API 프록시 | Next.js API Routes | API 키 보호, CORS 우회 |
| 캐싱 | Next.js ISR + Redis(선택) | 인구·상권은 월 1회 갱신이면 충분 |
| 배포 | Vercel | 무료 티어 |
| 버전관리 | GitHub (learn-slowly) | 기존 조직 |

**Supabase는 1차에서는 미사용** — API에서 직접 가져오고, 필요시 캐시 용도로 도입.

## 4. 주요 기능

### 4.1 지도 기반 시각화 (MVP)

- 행정동 경계 오버레이 + 선거구 색상 구분
- 인구 밀도 히트맵 (행정동별 주민등록 인구)
- 버스정류장 마커 (위치 + 경유 노선 수)
- 상가 밀집도 히트맵 (업종별 필터)
- 투표소 위치 + 선거인 수 표시
- 레이어 토글 UI

### 4.2 분석 기능 (Phase 2)

- 선거구별 인구 구성 (연령대·성별 피라미드)
- 교통 접근성 점수 (정류장 반경 500m 내 인구)
- 상권 활성도 ↔ 유동인구 추정
- 과거 선거 득표·투표율 패턴

### 4.3 실전 활용 (Phase 3)

- 유세 장소 추천 (복합 점수)
- 시간대별 가이드
- 인쇄용 선거구 요약표

## 7. 리스크 및 대응

| 리스크 | 대응 |
| --- | --- |
| API 응답 느림/장애 | Next.js ISR로 캐싱, 정적 fallback 데이터 준비 |

| API 일일 호출 한도 | 개발계정 10,000건 → 운영계정 전환 또는 캐싱 강화 |

## 8. 다른 도시 확장 시나리오

```typescript
// 새 도시 추가: config/cities/changwon.ts 한 파일만 생성
export const CHANGWON: CityConfig = {
  code: "38010",
  name: "창원시",
  province: "경상남도",
  center: [35.2280, 128.6811],
  zoom: 12,
  tagoCode: 38010,
  electionId: "20220601",
};
```

URL: `/changwon` → 창원시 데이터 자동 로딩. API 래퍼가 도시코드 기반이므로 코드 수정 없이 동작.
