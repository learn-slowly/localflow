# LocalFlow — 지역 선거 유동인구 분석 플랫폼

> **프로젝트명**: LocalFlow **목적**: 기초의원 선거 유세 전략 수립을 위한 유동인구·교통·상권·선거 데이터 종합 분석 플랫폼 **핵심 설계 원칙**: 도시코드만 교체하면 전국 어떤 시·군에도 적용 가능한 구조 **선거일**: 2026-06-03 (D-67) **작성일**: 2026-03-25 / **최종 업데이트**: 2026-03-27 **작성자**: 레고

---

## 1. 프로젝트 배경

- 2026년 6월 3일 제9회 전국동시지방선거 예정
- 진주시 기초의원(시의원) 선거에서 "어디에, 언제 사람이 모이는가"를 데이터로 파악
- 정의당 경남도당 차원의 유세 전략 수립 근거 자료
- **확장 목표**: 성공하면 경남 내 다른 시·군, 나아가 전국 적용

## 4. API 기반 데이터 아키텍처

### 4.1 설계 원칙

```
[설정 파일]          [API 레이어]           [시각화]
cityCode: 48170  →  각 API에 도시코드 전달  →  지도 렌더링
cityName: 진주시     데이터 정규화           레이어 토글
```

- 도시코드(행정표준코드) 하나로 모든 API 호출을 통일
- 다른 도시 적용 시 설정 파일의 코드만 변경

### 4.2 API 소스 목록 (전량 승인 완료)

#### 공공데이터포털 (data.go.kr) — 6건

| \# | 데이터 | API명 | 제공기관 |
| --- | --- | --- | --- |
| ① | **주민등록 인구** (행정동·통반 단위, 세대현황) | 행정안전부\_법정동별 주민등록 인구 및 세대현황 | 행정안전부 |
| ② | **상가(상권) 정보** (업소명, 업종, 좌표) | 소상공인시장진흥공단\_상가(상권)정보 | 소상공인시장진흥공단 |
| ③ | **투·개표 정보** (득표수, 투표율) | 중앙선거관리위원회\_투·개표 정보 | 중앙선관위 |
| ④ | **선거인수 정보** | 중앙선거관리위원회\_선거인수 정보 | 중앙선관위 |
| ⑤ | **투표소 정보** (직전 선거 기준) | 중앙선거관리위원회\_투표소 정보 | 중앙선관위 |
| ⑥ | **코드·후보자 정보** | 중앙선거관리위원회\_코드/후보자 정보 | 중앙선관위 |

#### 별도 사이트 — 5건

| \# | 데이터 | API명 | 사이트 |
| --- | --- | --- | --- |
| ⑦ | **행정동 경계** (GeoJSON) | 읍면동법정경계 | safetydata.go.kr |
| ⑧ | **교통 승하차·버스 노선·정류장** | 교통카드 빅데이터 OPEN API | stcis.go.kr |
| ⑨ | **카카오맵 JavaScript** | 지도 SDK | developers.kakao.com |
| ⑩ | **카카오 REST API** | 주소검색·좌표변환 | developers.kakao.com |
| ⑪ | **생활안전지도** | 주요시설 정보 (15종) | safemap.go.kr |

### 4.3 API 키 구조 (5종)

```env
DATA_GO_KR_API_KEY=         # 공공데이터포털 공통 (서버 전용, 6건)
BOUNDARY_API_KEY=           # 재난안전데이터 (서버 전용)
TRANSIT_API_KEY=            # 교통카드 빅데이터 (서버 전용)
NEXT_PUBLIC_KAKAO_MAP_KEY=  # 카카오맵 JavaScript (클라이언트)
KAKAO_REST_API_KEY=         # 카카오 REST API (서버 전용)
```
